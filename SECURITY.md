# Security Policy

## Supported Scope

Security reports are welcome for:

- Next.js application routes
- Appwrite function handlers
- inbox lookup and restore flows
- Mailgun inbound processing
- attachment handling

## Reporting a Vulnerability

Do not open public GitHub issues for security vulnerabilities.

Instead, report vulnerabilities privately to:

- `rahilisvahora@gmail.com`

Include:

- a description of the issue
- impact assessment
- reproduction steps
- proof of concept if available

## Secrets and Local Development

- Never commit production API keys or `.env` files.
- Use placeholder values in examples and documentation.
- Rotate any credential immediately if it was exposed in a commit, screenshot, log, or shared workspace.

## Response Expectations

Best effort review will be provided, but no formal SLA is guaranteed.
